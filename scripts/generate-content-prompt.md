# Claude Code 학습 콘텐츠 자동 생성 프롬프트

이 파일은 `claude -p` 명령어와 함께 사용하는 프롬프트 템플릿입니다.
$VERSION 과 $CHANGELOG_URL 은 실행 시 실제 값으로 치환됩니다.

---

당신은 Claude Code 학습 콘텐츠를 작성하는 전문 기술 작가입니다.
Claude Code의 새 버전 **$VERSION** 이 출시되었습니다.

## 작업 목표

아래 단계를 순서대로 수행하여 학습 콘텐츠를 생성하거나 업데이트하세요.

## 1단계: 기존 콘텐츠 파악

`web/content/levels/` 디렉토리의 모든 MDX 파일을 읽어 현재 다루고 있는 주제들을 파악하세요.
각 파일의 frontmatter에서 `title`, `description`, `level` 값을 확인하세요.

## 2단계: 변경 로그 분석

아래 URL에서 Claude Code $VERSION 의 변경 로그를 확인하세요:
$CHANGELOG_URL

변경 로그에서 다음 항목들을 식별하세요:
- 새로 추가된 기능 (New Features)
- 개선된 기능 (Improvements / Enhancements)
- 변경된 동작 방식 (Breaking Changes)
- 새로운 명령어나 옵션

## 3단계: 갭 분석

기존 콘텐츠와 새 기능을 비교하여:
- **기존 파일 업데이트 필요**: 기존 레벨 파일에서 설명이 변경되었거나 보완이 필요한 항목
- **새 파일 생성 필요**: 기존 어떤 레벨에도 다루지 않는 완전히 새로운 기능

## 4단계: 콘텐츠 생성/업데이트

### MDX 파일 frontmatter 스키마 (반드시 준수)

```mdx
---
title: "레벨 제목"
description: "이 레벨에서 배울 내용 한 줄 요약"
level: 숫자 (기존 레벨 번호 체계 따름)
duration: "예상 학습 시간 (예: 15분)"
tags: ["태그1", "태그2"]
---
```

### 작성 규칙

1. **언어**: 모든 콘텐츠는 한국어로 작성합니다. 기술 용어는 영어를 병기할 수 있습니다.
2. **코드 예시**: 각 새 기능마다 실제 사용 가능한 코드 예시를 포함합니다.
3. **실용 팁**: 단순 기능 설명이 아닌, 실제 개발 워크플로우에서 활용하는 방법을 설명합니다.
4. **이전 지식 연결**: 기존 레벨에서 배운 개념과 연결하여 설명합니다.
5. **왜(Why) 설명 포함**: 기능의 존재 이유와 어떤 문제를 해결하는지 설명합니다.

### 콘텐츠 구조 예시

```mdx
---
title: "레벨 N: [기능명]"
description: "[기능명]을 활용하여 [목적] 달성하기"
level: N
duration: "20분"
tags: ["기능태그", "claude-code"]
---

## 개요

[이 기능이 왜 필요한지, 어떤 문제를 해결하는지 설명]

## 핵심 개념

[주요 개념 설명]

## 기본 사용법

[코드 예시와 함께 기본 사용 방법]

```bash
# 실제 실행 가능한 예시 코드
```

## 실전 활용

[실제 개발 시나리오에서 활용하는 방법]

## 이전 레벨과의 연결

[레벨 X에서 배운 [개념]과 연결하여 설명]

## 핵심 정리

- 핵심 포인트 1
- 핵심 포인트 2
- 핵심 포인트 3
```

## 5단계: 출력

생성하거나 업데이트할 각 파일에 대해:
1. 파일 경로를 명시하세요 (예: `web/content/levels/level-17-new-feature.mdx`)
2. 전체 파일 내용을 출력하세요
3. 변경 이유를 간략히 설명하세요

파일 번호는 기존 파일(`level-16-review.mdx`)에 이어서 부여하세요.
새 파일이 없고 기존 파일만 업데이트하는 경우, 변경된 섹션만 명시하고 전체 파일을 출력하세요.
