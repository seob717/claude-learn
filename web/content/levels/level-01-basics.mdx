## 바이브 코딩(Vibe Coding)이란?

2025년 Andrej Karpathy가 제시한 개념으로, AI에게 자연어로 지시하여 코드를 작성하는 프로그래밍 방식이다.
전통적 코딩이 "한 줄 한 줄 직접 작성"이라면, 바이브 코딩은 "무엇을 만들지 설명하고 AI가 구현"하는 접근법이다.

```
전통적 코딩:
  개발자 → 코드 직접 작성 → 컴파일/실행 → 디버깅

바이브 코딩:
  개발자 → 자연어 지시 → AI가 코드 생성 → 개발자가 검토/피드백 → AI가 수정
```

### 바이브 코딩의 핵심 원칙

- **자연어 우선**: 코드 문법보다 "무엇을 원하는지"를 명확히 표현
- **반복적 대화**: 한 번에 완벽한 결과를 기대하지 않고 점진적으로 개선
- **검토 필수**: AI가 생성한 코드를 항상 이해하고 검증

> Claude Code는 바이브 코딩을 위해 설계된 도구다. 이 가이드 전체에서 바이브 코딩의 원칙이 적용된다.

## 코드 에이전트(Code Agent)란?

단순한 코드 자동완성이 아닌, **자율적으로 판단하고 행동**하는 AI 도구를 말한다.

| 구분 | 코드 자동완성 | 코드 에이전트 |
|------|-------------|-------------|
| 동작 방식 | 다음 줄 예측 | 목표를 이해하고 계획 수립 |
| 도구 사용 | 없음 | 파일 읽기/쓰기, 명령어 실행, 검색 |
| 범위 | 현재 파일 한 줄 | 프로젝트 전체 |
| 자율성 | 없음 (사용자가 트리거) | 높음 (스스로 다음 행동 결정) |
| 예시 | GitHub Copilot 인라인 | Claude Code, Cursor Agent |

Claude Code는 에이전틱 루프(아래에서 설명)를 통해 코드 에이전트로 동작한다.

## Claude Code란?

Anthropic이 만든 **에이전틱 코딩 도구**. 코드 읽기, 편집, 명령어 실행, 개발 도구 연동을 자동으로 수행한다.

## 작동 원리 (에이전틱 루프)

```
사용자 프롬프트 → 도구 선택 → 도구 실행 → 결과 관찰 → 다음 액션 결정 → ... → 완료
```

사용 가능한 도구: `Bash`, `Read`, `Edit`, `Write`, `Grep`, `Glob`, `WebFetch`, `WebSearch`, `Task` 등

## 설치

```bash
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash

# Homebrew
brew install --cask claude-code

# 업데이트
claude update
```

## 기본 사용법

```bash
claude                         # 대화형 세션 시작
claude "이 프로젝트 설명해줘"    # 초기 프롬프트와 함께 시작
claude -p "query"              # 비대화형(headless) 모드 — 답변 후 종료
claude -c                      # 가장 최근 대화 이어하기
claude --resume auth-refactor  # 이름으로 세션 재개
```

## 빠른 접두사

| 접두사 | 동작 |
|--------|------|
| `!` | bash 명령어 직접 실행 (예: `!ls -la`) |
| `@` | 파일 경로 자동완성 |
| `/` | 슬래시 커맨드/스킬 메뉴 |
