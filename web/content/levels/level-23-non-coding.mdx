## 코딩 외 활용 개요

많은 사람들이 Claude Code를 코드 작성 도구로만 생각하지만, 실제로는 훨씬 다양한 작업에 활용할 수 있다.

```
Claude Code가 잘하는 비코딩 작업:

문서 작성    → README, API 문서, 기술 블로그 초안
데이터 분석  → CSV 분석, 로그 파싱, 패턴 추출
프로젝트 관리 → PR 설명, 이슈 분류, 마이그레이션 계획
학습 및 교육 → 코드베이스 이해, 개념 설명, 아키텍처 다이어그램
시스템 관리  → 서버 설정, Docker/K8s 구성, 쉘 스크립트
커뮤니케이션 → 기술 제안서, 회의록 정리, 이메일 초안
```

Claude Code는 파일 시스템에 접근하고 명령어를 실행할 수 있기 때문에, 터미널 환경에서 이루어지는 거의 모든 지식 작업에 적용할 수 있다.

## 문서 작성 및 관리

### README.md 자동 생성

```bash
# 프로젝트 구조를 분석하고 README를 생성
claude --tools "Read,Glob,Bash(git log --oneline -20)" \
  -p "
    이 프로젝트의 package.json, 주요 소스 파일, git 히스토리를 읽고
    다음 항목을 포함한 README.md를 작성해줘:
    - 프로젝트 소개 (한 줄 설명)
    - 주요 기능 목록
    - 설치 방법
    - 사용 방법 (예시 포함)
    - 기술 스택
    - 라이센스
  "
```

### API 문서 업데이트

```bash
# 소스 코드를 읽고 API 문서를 최신화
claude --tools "Read,Glob,Write" \
  -p "
    src/routes/ 폴더의 모든 API 엔드포인트를 읽고
    docs/api.md 파일을 OpenAPI 스타일로 업데이트해줘.
    각 엔드포인트에 대해 메서드, 경로, 파라미터, 응답 예시를 포함해줘.
  "
```

### CHANGELOG 작성

```bash
# git 히스토리를 기반으로 CHANGELOG 생성
claude --tools "Bash(git log --oneline v1.0.0..HEAD)" \
  -p "
    위 git 커밋 목록을 분석해서 CHANGELOG.md를 작성해줘.
    다음 카테고리로 분류해줘:
    - 새로운 기능 (feat)
    - 버그 수정 (fix)
    - 성능 개선 (perf)
    - 주요 변경 (breaking change)
    Keep a Changelog 형식을 따라줘.
  "
```

### 기술 블로그 초안

```bash
# 구현한 기능을 기술 블로그 포스트로 변환
claude --tools "Read" \
  -p "
    src/features/payment/ 폴더의 코드를 읽고
    '결제 시스템 구현기' 제목의 기술 블로그 초안을 써줘.
    독자는 백엔드 개발자이고, 다음을 포함해줘:
    - 왜 이 방식을 선택했는지
    - 핵심 아키텍처 결정
    - 어려웠던 점과 해결 방법
    - 코드 예시 (핵심 부분만)
    분량: 1500~2000자
  "
```

## 데이터 분석

### CSV/JSON 데이터 분석

```bash
# 매출 데이터 분석
claude --tools "Read,Bash(python3 -c 'import csv; ...')" \
  -p "
    sales-2026-q1.csv 파일을 분석해줘:
    - 월별 총 매출
    - 상위 10개 제품
    - 전월 대비 성장률
    - 이상치가 있는 데이터 행
    결과를 표 형태로 보여줘.
  "

# JSON 데이터 구조 분석
claude --tools "Read,Bash(jq '.' data.json | head -50)" \
  -p "
    data.json의 구조를 분석하고 다음을 알려줘:
    - 데이터 스키마 (필드명과 타입)
    - 총 레코드 수
    - null 값이 많은 필드
    - 데이터 품질 이슈
  "
```

### 로그 파일 분석

```bash
# 서버 에러 로그 분석
claude --tools "Bash(tail -n 1000 /var/log/app.log)" \
  -p "
    최근 1000줄의 애플리케이션 로그를 분석해줘:
    - 에러 유형별 빈도 (상위 5개)
    - 에러가 집중된 시간대
    - 가장 자주 실패하는 엔드포인트
    - 즉시 조치가 필요한 심각한 에러
  "
```

### 패턴 추출 및 통계

```bash
# 코드베이스 통계 추출
claude --tools "Bash(find src -name '*.ts' | wc -l)", \
       "Bash(wc -l src/**/*.ts | tail -1)", \
       "Bash(grep -r 'TODO' src/ | wc -l)" \
  -p "
    위 명령어 결과를 바탕으로 프로젝트 현황 리포트를 만들어줘:
    - 파일 수, 코드 라인 수
    - TODO 항목 수와 위치
    - 가장 큰 파일 목록
    개발팀 주간 미팅에서 공유할 형식으로 작성해줘.
  "
```

## 프로젝트 관리

### 코드 리뷰 요약

```bash
# PR의 변경 사항을 읽기 쉽게 요약
claude --tools "Bash(git diff main...feature-branch)" \
  -p "
    이 PR의 변경 사항을 비기술적인 팀원도 이해할 수 있게 요약해줘:
    - 이 PR이 해결하는 문제
    - 주요 변경 내용 (3~5줄)
    - 테스트 방법
    - 주의해서 봐야 할 부분
  "
```

### PR 설명 자동 작성

```bash
# 구현 내용을 바탕으로 PR 설명 생성
claude --tools "Bash(git diff main --stat)", \
       "Bash(git log main..HEAD --oneline)" \
  -p "
    다음 형식으로 GitHub PR 설명을 작성해줘:
    ## 변경 사항
    (무엇을 왜 변경했는지)

    ## 테스트
    - [ ] 단위 테스트 통과
    - [ ] 수동 테스트 시나리오

    ## 스크린샷
    (UI 변경이 있다면 여기에 추가)

    ## 체크리스트
    - [ ] 코드 리뷰 완료
    - [ ] 문서 업데이트
  "
```

### 이슈 분류 및 우선순위 제안

```bash
# 백로그 이슈를 분석하고 우선순위 제안
claude --tools "Read(issues-export.json)" \
  -p "
    이 이슈 목록을 분석해서 우선순위를 제안해줘.
    다음 기준으로 분류해줘:
    - P0 (즉시 해결): 서비스 중단, 데이터 손실 위험
    - P1 (이번 스프린트): 주요 기능 버그, 사용자 영향 큼
    - P2 (다음 스프린트): 개선 사항, 사용자 영향 중간
    - P3 (백로그): 작은 개선, 내부 기술 부채
    각 이슈에 예상 작업 시간도 추가해줘.
  "
```

### 마이그레이션 계획 수립

```bash
# 기존 코드를 분석해서 마이그레이션 계획 작성
claude --tools "Read,Glob,Bash(npm list --depth=0)" \
  -p "
    현재 프로젝트를 분석하고 Node.js 18 → 20 업그레이드 계획을 세워줘:
    - 현재 의존성 중 호환성 문제가 예상되는 것
    - 마이그레이션 단계 (순서대로)
    - 각 단계별 예상 소요 시간
    - 롤백 계획
    - 테스트 체크리스트
  "
```

## 학습 및 교육

### 새 코드베이스 학습

```bash
# 처음 보는 프로젝트를 빠르게 파악
claude --tools "Read,Glob" \
  -p "
    이 프로젝트를 처음 보는 개발자에게 설명하듯이 알려줘:
    - 이 프로젝트가 무엇을 하는지
    - 핵심 모듈과 각 역할
    - 데이터 흐름 (요청이 들어와서 응답이 나가는 경로)
    - 코드 작성 시 알아야 할 규칙과 패턴
    - 개발 환경 세팅 방법
  "
```

### 기술 개념 설명 요청

```bash
# 코드를 읽으면서 모르는 패턴 이해
claude --tools "Read(src/middleware/auth.ts)" \
  -p "
    이 인증 미들웨어 코드를 읽고 설명해줘.
    특히 다음을 이해하고 싶어:
    - JWT 토큰 검증이 어떻게 작동하는지
    - refresh token 로직
    - 보안상 주의한 부분
    비유를 들어서 쉽게 설명해줘.
  "
```

### 아키텍처 다이어그램 (텍스트 기반)

```bash
# 코드에서 아키텍처 다이어그램 생성
claude --tools "Read,Glob" \
  -p "
    이 프로젝트의 아키텍처를 ASCII 다이어그램으로 그려줘.
    다음을 포함해줘:
    - 클라이언트 → 서버 → DB 흐름
    - 주요 서비스 컴포넌트
    - 외부 서비스 연동 (API, 메시지 큐 등)
    Mermaid 다이어그램 형식도 함께 만들어줘.
  "
```

### 코드 리뷰 학습

```bash
# 작성한 코드에서 개선점 배우기
claude --tools "Read(src/services/UserService.ts)" \
  -p "
    내가 작성한 이 코드를 시니어 개발자 관점에서 리뷰해줘.
    단순히 문제점만 지적하지 말고, 왜 그것이 문제인지,
    어떻게 개선해야 하는지, 더 나은 패턴은 무엇인지 설명해줘.
    내가 배울 수 있는 개념이나 패턴이 있다면 함께 알려줘.
  "
```

## 시스템 관리

### 서버 설정 파일 분석/생성

```bash
# Nginx 설정 생성
claude -p "
  다음 요구사항에 맞는 Nginx 설정 파일을 작성해줘:
  - 도메인: api.myapp.com
  - 백엔드: localhost:3000
  - HTTPS 강제 리다이렉트
  - CORS 헤더 (허용 도메인: myapp.com, admin.myapp.com)
  - 요청 크기 제한: 10MB
  - gzip 압축 활성화
  - 접근 로그: /var/log/nginx/api-access.log
  보안 관련 헤더도 포함해줘.
"
```

### Docker/Kubernetes 설정

```bash
# 기존 앱을 컨테이너화
claude --tools "Read(package.json)", "Read(src/index.ts)" \
  -p "
    이 Node.js 앱을 컨테이너화하기 위한 Dockerfile과
    docker-compose.yml을 작성해줘.
    다음을 고려해줘:
    - 멀티 스테이지 빌드로 이미지 크기 최소화
    - 개발/프로덕션 환경 분리
    - 환경변수 관리 (.env 파일 사용)
    - 헬스체크 설정
    - 로그 볼륨 마운트
  "
```

### 쉘 스크립트 작성 및 디버깅

```bash
# 복잡한 배포 스크립트 작성
claude -p "
  다음 작업을 자동화하는 bash 배포 스크립트를 작성해줘:
  1. git pull로 최신 코드 가져오기
  2. npm ci로 의존성 설치
  3. npm run build로 빌드
  4. pm2로 서버 재시작 (무중단 배포)
  5. 헬스체크 (30초 내로 응답 없으면 롤백)
  6. Slack에 배포 결과 알림

  에러 처리와 롤백 로직을 반드시 포함하고,
  각 단계에 진행 상황을 출력해줘.
"
```

## 커뮤니케이션

### 기술 제안서 초안

```bash
# 기술적 결정을 비기술 팀원에게 설명하는 문서 작성
claude --tools "Read(architecture-draft.md)" \
  -p "
    이 기술 아키텍처 초안을 바탕으로 경영진 보고용 제안서를 작성해줘.
    기술 용어를 최대한 쉽게 풀어서 설명하고,
    다음 항목을 포함해줘:
    - 현재 문제점 (비즈니스 영향 중심)
    - 제안 솔루션 (기술적 세부사항 최소화)
    - 예상 비용과 개발 기간
    - 기대 효과 (숫자로)
    - 리스크와 대응 방안
    A4 2페이지 분량으로 작성해줘.
  "
```

### 이메일/슬랙 메시지 작성

```bash
# 장애 보고 이메일 작성
claude --tools "Read(incident-log.txt)" \
  -p "
    이 장애 로그를 바탕으로 고객에게 보낼 장애 보고 이메일을 작성해줘.
    포함할 내용:
    - 발생 시간과 복구 시간
    - 영향 범위 (기능, 사용자 수)
    - 원인 (기술적 세부사항 없이)
    - 재발 방지 대책
    - 사과 인사

    전문적이되 인간적인 톤으로 작성해줘.
    영어 버전도 함께 만들어줘.
  "
```

## 실습 예시

### 실습 1: 프로젝트 건강 리포트 생성

```bash
# 프로젝트의 현재 상태를 종합 리포트로 작성
claude --tools \
  "Bash(git log --oneline -30)", \
  "Bash(npm run test -- --coverage 2>&1 | tail -20)", \
  "Bash(npx eslint src/ --format json 2>&1 | python3 -c 'import json,sys; d=json.load(sys.stdin); print(sum(len(f[\"messages\"]) for f in d))')", \
  "Bash(find src -name '*.ts' | xargs wc -l | tail -1)" \
  -p "
    위 정보를 바탕으로 이번 주 프로젝트 건강 리포트를 작성해줘:
    - 코드 커버리지 현황
    - 린트 에러 수
    - 최근 커밋 활동
    - 개선이 필요한 부분 (우선순위 순)
    팀 슬랙에 공유할 형식으로 만들어줘.
  "

# 예상 결과:
# 📊 이번 주 프로젝트 건강 리포트
# ✅ 테스트 커버리지: 78% (목표 80% 미달)
# ⚠️ 린트 에러: 12개 (지난주 대비 +3)
# 🚀 커밋 활동: 15건 (활발)
# ...
```

### 실습 3: 기술 부채 분석 리포트

```bash
# 코드베이스의 기술 부채를 분석
claude --tools \
  "Bash(grep -r 'TODO\|FIXME\|HACK\|XXX' src/ | wc -l)", \
  "Bash(grep -r 'any' src/**/*.ts | wc -l)", \

## 핵심 정리

```
Claude Code 비코딩 활용 요약:

문서화     → README, API 문서, CHANGELOG, 블로그 초안
           → --tools "Read,Glob,Write" 활용

데이터     → CSV/JSON 분석, 로그 파싱, 통계
           → --tools "Bash(python3/jq)" 활용

프로젝트   → PR 설명, 이슈 분류, 마이그레이션 계획
           → --tools "Bash(git ...)" 활용

학습       → 코드 이해, 개념 설명, 아키텍처 파악
           → --tools "Read,Glob" 활용

시스템     → 설정 파일 생성, Docker, 쉘 스크립트
           → --tools "Read,Write,Bash" 활용

커뮤니케이션 → 제안서, 이메일, 장애 보고서
             → 텍스트 입력 또는 --tools "Read" 활용

핵심 원칙:
  구체적인 요구사항 명시 → 더 좋은 결과

  대상 독자 명시 → 적절한 톤과 수준 조절
