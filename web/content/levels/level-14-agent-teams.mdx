## 개요

여러 Claude Code 인스턴스가 **공유 태스크 리스트**로 협업하는 시스템.

**활성화:**

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

## 구조

```
Lead Agent (리더)
├── Teammate 1 (실행자)
├── Teammate 2 (실행자)
└── Teammate 3 (실행자)
    └── 공유 태스크 리스트
```

- 리더가 태스크를 생성하고 팀원에게 분배
- 팀원은 독립적인 컨텍스트 윈도우를 가짐
- 공유 태스크 리스트와 메시징으로 소통

## 표시 모드

| 모드 | 설명 |
|------|------|
| `in-process` | 메인 프로세스 내에서 표시 |
| `tmux` | tmux 분할 패널에서 표시 |
| `auto` | 환경에 따라 자동 선택 |

## 관련 Hook 이벤트

- `TeammateIdle`: 팀원 유휴 시
- `TaskCompleted`: 태스크 완료 시

## Multiple Claude 인스턴스

Agent Teams와 별개로, **여러 Claude Code 프로세스를 독립적으로** 실행하여 병렬 작업할 수 있다.

### 터미널 분할 방식

```bash
# tmux로 2개 Claude 인스턴스 동시 실행
tmux new-session -d -s multi

# 좌측: API 관련 작업
tmux send-keys -t multi "cd ~/project && claude --worktree api-work" Enter

# 우측: UI 관련 작업
tmux split-window -h
tmux send-keys -t multi "cd ~/project && claude --worktree ui-work" Enter

tmux attach -t multi
```

### Worktree + 다중 인스턴스

```bash
# 각 인스턴스가 독립적인 worktree에서 작업
# 인스턴스 1: 백엔드 리팩토링
claude --worktree backend-refactor -p "API 라우터를 Express에서 Hono로 마이그레이션"

# 인스턴스 2: 프론트엔드 리팩토링 (동시에)
claude --worktree frontend-refactor -p "클래스 컴포넌트를 함수형으로 전환"

# 인스턴스 3: 테스트 보강 (동시에)
claude --worktree test-coverage -p "커버리지 80% 달성을 위해 테스트 추가"
```

### Multiple Claude vs Agent Teams

| 비교 | Multiple Claude | Agent Teams |
|------|----------------|-------------|
| 소통 방식 | 독립적 (소통 없음) | 공유 태스크 리스트 |
| 파일 충돌 | Worktree로 격리 필요 | 팀 내 조율 |
| 적합한 작업 | 완전히 독립적인 작업 | 상호 의존적인 작업 |
| 설정 | 추가 설정 불필요 | 환경변수 필요 |

## Claude Cowork

**Claude Cowork**은 claude.ai 웹에서 시작한 작업을 로컬 Claude Code로 이어받는 기능이다.

### 워크플로

```
1. claude.ai에서 설계/계획 논의
     ↓
2. /teleport 또는 /desktop 으로 로컬 전환
     ↓
3. 로컬 Claude Code에서 실제 구현
     ↓
4. 결과를 다시 claude.ai에서 리뷰
```

### 관련 커맨드

```bash
# claude.ai 세션을 로컬로 가져오기
/teleport

# CLI 세션을 데스크톱 앱으로 핸드오프
/desktop
```

### 활용 시나리오

```
시나리오 1: 아키텍처 설계 → 구현
  claude.ai에서 시스템 설계 논의 → /teleport → 로컬에서 코드 작성

시나리오 2: 코드 리뷰 → 수정
  claude.ai에서 코드 리뷰 → 수정 사항 정리 → /teleport → 수정 실행

시나리오 3: 팀 협업
  팀원 A: claude.ai에서 설계 → /teleport → 구현
  팀원 B: 별도 세션에서 다른 기능 구현
  → 각자 독립적으로 작업 후 git에서 통합
```

> Cowork은 claude.ai의 **긴 컨텍스트와 멀티모달 능력**과 Claude Code의 **파일 시스템 접근과 도구 실행 능력**을 결합한다.
