## Sandbox 모드란?

Sandbox는 Claude Code가 실행하는 모든 명령과 코드를 **Docker 컨테이너 안에 격리**시키는 보안 실행 환경이다.
호스트 시스템에 직접 접근하지 못하므로, 신뢰하기 어려운 코드나 실험적인 작업을 안전하게 수행할 수 있다.

### Sandbox가 필요한 상황

| 상황 | 이유 |
|------|------|
| 외부 저장소 코드 분석 | 악성 스크립트 실행 위험 차단 |
| 의존성 설치 실험 | 호스트 환경 오염 방지 |
| CI/CD 파이프라인 | 재현 가능하고 격리된 빌드 환경 |
| 보안 감사 | 취약점 탐색 중 시스템 보호 |
| 팀 공용 서버 | 여러 사용자 간 격리 보장 |

## `--sandbox` 플래그 사용법

```bash
# 기본 Sandbox 모드 실행
claude --sandbox

# 비대화형 Sandbox 실행
claude --sandbox -p "이 스크립트를 실행하고 결과를 알려줘"

# 특정 모델 + Sandbox
claude --sandbox --model claude-opus-4-5

# 권한 모드와 함께
claude --sandbox --permission-mode plan
```

### 설정 파일로 항상 Sandbox 활성화

```json
// ~/.claude/settings.json 또는 .claude/settings.json
{
  "sandbox": true
}
```

> 프로젝트 단위로 Sandbox를 강제할 때 `.claude/settings.json`에 설정한다.
> 팀 전체에 적용하려면 이 파일을 git에 커밋한다.

## Docker 기반 격리 구조

```
┌─────────────────────────────────────────┐
│            호스트 시스템                  │
│                                         │
│  Claude Code CLI                        │
│       │                                 │
│       ▼                                 │
│  ┌─────────────────────────────────┐    │
│  │        Docker Container          │    │
│  │                                 │    │
│  │  - 파일 시스템: 읽기 전용 마운트  │    │
│  │  - 네트워크: 격리 또는 차단      │    │
│  │  - 프로세스: 컨테이너 내 한정    │    │
│  │  - 권한: 최소 권한 원칙 적용     │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### 격리되는 리소스

- **파일 시스템**: 지정된 디렉토리만 마운트 (기본적으로 현재 프로젝트)
- **네트워크**: 외부 인터넷 접근 차단 가능
- **프로세스**: 컨테이너 내부로만 한정
- **환경 변수**: 호스트 env 노출 최소화

## `--fallback-model` 옵션

레이트 리밋(rate limit)에 도달하거나 기본 모델을 사용할 수 없을 때 자동으로 전환할 **대체 모델**을 지정한다.

```bash
# opus가 한도 초과 시 sonnet으로 자동 전환
claude --model claude-opus-4-5 --fallback-model claude-sonnet-4-5

# 비대화형 파이프라인에서 특히 유용
claude -p "대규모 코드 분석" \
  --model claude-opus-4-5 \
  --fallback-model claude-haiku-4-5 \
  --output-format json
```

### Fallback 동작 순서

```
1. 기본 모델(--model)로 요청 시도
2. 레이트 리밋 / 503 오류 발생
3. --fallback-model로 자동 재시도
4. fallback도 실패 시 오류 반환
```

### 권장 Fallback 조합

| 기본 모델 | 권장 Fallback | 용도 |
|-----------|--------------|------|
| claude-opus-4-5 | claude-sonnet-4-5 | 고품질 분석 |
| claude-sonnet-4-5 | claude-haiku-4-5 | 일반 작업 |
| claude-opus-4-5 | claude-haiku-4-5 | 비용 최적화 |

## `claude security scan` 보안 취약점 스캔

프로젝트 코드에서 보안 취약점을 자동으로 탐지한다.

```bash
# 현재 디렉토리 전체 스캔
claude security scan

# 특정 파일/디렉토리 스캔
claude security scan src/

# JSON 출력 (CI 파이프라인 연동)
claude security scan --output-format json > security-report.json

# 심각도 필터링
claude security scan --severity high
```

### 탐지 가능한 취약점 유형

```
- SQL Injection
- XSS (Cross-Site Scripting)
- 하드코딩된 시크릿 (API 키, 비밀번호)
- 안전하지 않은 의존성
- 권한 취약점 (path traversal 등)
- 안전하지 않은 암호화 사용
```

### CI/CD 파이프라인 연동 예시

```yaml
# .github/workflows/security.yml
name: Security Scan
on: [push, pull_request]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Claude Security Scan
        run: |
          claude security scan --output-format json > report.json
          # 고위험 취약점 발견 시 빌드 실패
          claude security scan --severity critical --fail-on-findings
```

## 네트워크 격리 설정

Sandbox 환경에서 네트워크 접근을 세밀하게 제어할 수 있다.

```json
// .claude/settings.json
{
  "sandbox": true,
  "sandboxConfig": {
    "network": "none",          // 완전 차단
    "allowedHosts": [           // 특정 호스트만 허용
      "registry.npmjs.org",
      "pypi.org"
    ],
    "readOnlyPaths": ["/etc"],  // 읽기 전용 경로
    "writablePaths": ["/tmp"]   // 쓰기 허용 경로
  }
}
```

### 네트워크 모드 옵션

| 모드 | 설명 | 사용 시나리오 |
|------|------|--------------|
| `none` | 완전 오프라인 | 민감한 코드 분석 |
| `restricted` | allowedHosts만 허용 | 패키지 설치 허용 |
| `full` | 제한 없음 (기본값) | 일반 개발 |

## 실습 예시

### 실습 1: 외부 코드 안전 분석

```bash
# 외부에서 받은 스크립트를 Sandbox에서 안전하게 분석
git clone https://github.com/unknown/project.git /tmp/review-project
cd /tmp/review-project

# Sandbox + 보안 스캔 조합
claude --sandbox -p "이 코드베이스의 보안 취약점과 악성 코드 가능성을 분석해줘"
claude security scan .
```

### 실습 2: Fallback이 있는 자동화 파이프라인

```bash
#!/bin/bash
# analyze.sh - 레이트 리밋에 강한 분석 스크립트

files=$(find src -name "*.ts" | head -20)

for file in $files; do
  claude --sandbox \
    --model claude-opus-4-5 \
    --fallback-model claude-sonnet-4-5 \
    -p "다음 파일의 코드 품질을 분석해줘: $file" \
    --output-format json >> analysis.jsonl
done
```

### 실습 3: 보안 스캔 + 자동 수정

```bash
# 취약점 스캔 후 Claude에게 수정 요청
claude security scan --output-format json > vulns.json

# 스캔 결과를 컨텍스트로 제공하여 수정
claude --sandbox \
  --allowedTools "Edit,Read" \
  -p "vulns.json의 취약점들을 수정해줘. 각 수정 후 설명을 달아줘"
```

## Sandbox와 Hooks 조합

```json
// .claude/settings.json
{
  "sandbox": true,
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "echo '[Sandbox] Bash 명령 실행: ' >> /tmp/sandbox-audit.log"
      }]
    }]
  }
}
```

> Sandbox + Hooks 조합으로 감사 로그(audit log)를 남길 수 있다.
> 보안이 중요한 프로젝트에서는 모든 Bash 실행을 기록하는 것이 좋다.

## 핵심 정리

```
--sandbox          → Docker 격리 실행
--fallback-model   → 레이트 리밋 시 대체 모델 자동 전환
security scan      → 코드 보안 취약점 자동 탐지
sandboxConfig      → 네트워크·파일 시스템 세밀한 격리 설정
```
