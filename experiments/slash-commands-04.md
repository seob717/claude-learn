# 슬래시 커맨드 실습 가이드
# 목적: Level 4 - 슬래시 커맨드를 직접 체험하고 핵심 사용법을 익힌다

---

## 실습 1: 정보 확인 커맨드

아래 커맨드를 Claude Code 세션에서 직접 입력해보세요:

```
/status    → 버전, 모델, 계정, 연결 상태 확인
/cost      → 현재 세션의 토큰 사용량 확인
/context   → 컨텍스트 사용량을 컬러 그리드로 시각화
/usage     → 플랜 한도, 레이트 리밋 확인
```

### 왜 중요한가?
- `/cost`와 `/context`는 "지금 얼마나 컨텍스트를 썼나" 파악하는 핵심 도구
- 컨텍스트가 가득 차면 자동 압축이 일어나므로, 주기적 확인이 좋다

---

## 실습 2: 세션 관리 커맨드

```
/rename slash-cmd-practice   → 현재 세션에 이름 붙이기
/compact                     → 컨텍스트 수동 압축
/compact API 관련만 유지       → 특정 주제 포커스 압축
/clear                       → 대화 초기화 (세션은 유지)
```

### 왜 중요한가?
- `/rename`으로 이름을 붙이면 나중에 `claude -r slash-cmd-practice`로 재개 가능
- `/compact`는 긴 작업 중 컨텍스트 절약에 필수

---

## 실습 3: 모델과 모드 전환

```
/model          → 모델 선택 (←→ 키로 노력 수준 조절!)
/fast           → 빠른 모드 토글 (같은 모델, 2.5배 속도)
/plan           → 플랜 모드 진입 (읽기 전용 탐색)
```

### 핵심 포인트
- `/model` 선택 화면에서 좌우 화살표를 누르면 effort level이 바뀜
- `/fast`는 모델을 바꾸는 게 아니라 같은 모델의 출력 속도를 높이는 것
- `/plan`은 코드 수정 없이 탐색만 하고 싶을 때 사용

---

## 실습 4: 유틸리티 커맨드

```
/mcp            → 연결된 MCP 서버 상태 확인
/memory         → CLAUDE.md 메모리 편집
/export         → 대화 내보내기
/doctor         → 설치 상태 진단
```

---

## 실습 5: 되감기와 세션 재개

```
Esc Esc         → 이전 시점으로 되감기 (대화 + 코드)
/resume         → 이전 세션 목록에서 선택하여 재개
/rewind         → 특정 시점으로 되감기
```

### 왜 중요한가?
- Claude가 잘못된 방향으로 갔을 때 `Esc Esc`로 빠르게 복구
- 되감기는 대화뿐 아니라 코드 변경도 되돌린다 (git checkpoint)
