# Level 9: 세션 관리와 컨텍스트 실습 노트

> 목적: 세션 생명주기, 자동 압축, 자동 메모리의 동작 원리를 이해한다

---

## 핵심 개념

### 세션 생명주기

```
시작 → CLAUDE.md/rules 로드 → 대화 → (컨텍스트 가득) → 자동 압축 → 계속 → 종료
 │                                                                        │
 └─── claude -c (최근) 또는 /resume (선택) 또는 claude -r name 으로 재개 ←─┘
```

### 컨텍스트 윈도우 = Claude의 "작업 기억"
- 대화가 쌓이면 컨텍스트 윈도우가 차오른다
- 가득 차면 **자동 압축(auto-compaction)** 발생
- 핵심 정보는 보존, 나머지는 요약으로 교체

## 실습 결과

### 1. 세션 저장 구조 (이 프로젝트)

```
~/.claude/projects/-Users-yuseobshim-WebstormProjects-run/
├── {세션ID}.jsonl                ← 대화 로그 (JSONL 형식)
├── {세션ID}/
│   ├── subagents/                ← 서브에이전트 실행 로그
│   │   └── agent-acompact-*.jsonl   ← "compact" = 자동 압축 흔적!
│   └── tool-results/             ← 도구 실행 결과 캐시
│       └── toolu_*.txt
```

**발견**: `agent-acompact-` 접두사가 붙은 파일 = 자동 압축이 실행된 증거.
이전 학습 세션에서 대화가 길어져 압축이 발생했음을 확인.

### 2. 자동 메모리

저장 위치: `~/.claude/projects/<프로젝트경로>/memory/`

```
memory/
├── MEMORY.md          ← 인덱스 (처음 200줄 자동 로드)
├── debugging.md       ← 주제별 상세
└── patterns.md        ← 주제별 상세
```

- Claude가 반복 패턴 감지 시 자동 생성
- MEMORY.md 처음 200줄은 **매 세션 시작 시 자동 로드**
- 200줄 넘으면 잘리므로 간결하게 유지해야 함
- `/memory` 커맨드로 토글 가능

## 세션 관리 명령어 정리

### 세션 재개

| 방법 | 용도 | 예시 |
|------|------|------|
| `claude -c` | 가장 최근 세션 | 빠른 이어하기 |
| `claude -r <이름>` | 이름으로 재개 | `claude -r auth-refactor` |
| `claude --from-pr 123` | PR 연결 세션 | PR 기반 작업 이어하기 |
| `/resume` | 대화형 피커 | P: 미리보기, R: 이름변경, B: 브랜치 필터 |
| `/rename <이름>` | 현재 세션 이름 지정 | 나중에 찾기 쉽게 |

### 컨텍스트 관리

| 명령어 | 동작 |
|--------|------|
| `/compact` | 수동 압축 (공간 확보) |
| `/compact API 관련 유지` | 특정 주제 포커스 압축 |
| `/context` | 컬러 그리드로 사용량 시각화 |
| `/clear` | 대화 초기화 (세션은 유지) |
| `/export` | 대화를 파일로 내보내기 |
| `Esc Esc` | 되감기 (대화 + 코드 모두 이전 시점으로) |

### 자동 압축 vs 수동 압축

| 구분 | 자동 | 수동 (/compact) |
|------|------|-----------------|
| 발생 시점 | 컨텍스트 가득 참 | 사용자가 직접 실행 |
| 포커스 | 전체 대화 요약 | 지시어로 특정 주제 보존 가능 |
| 제어 | 불가 | 가능 |

## 체크포인트와 되감기

### 동작 원리
1. Claude가 파일을 변경할 때마다 git 체크포인트 자동 생성
2. `Esc Esc` 누르면 되감기 메뉴 표시
3. 선택한 시점으로 **대화 + 파일 시스템** 모두 복원

```
시점 A (파일 수정) → 시점 B (더 수정) → 시점 C (잘못됨!)
                                          │
                                     Esc Esc 누르면
                                          │
                                     시점 A 또는 B로 돌아감
                                     (대화도, 코드도 함께)
```

→ Level 5에서 배운 `Esc Esc` 단축키의 실제 동작 원리!

### Prompt Suggestions
응답 후 회색으로 나타나는 제안 프롬프트:
- `Tab`: 수락
- `Enter`: 수락 + 바로 제출
- **프롬프트 캐시 재활용** → 비용 최소화

## 이전 레벨과의 연결

| 레벨 | 연결점 |
|------|--------|
| Level 1 | `claude -c`, `--resume`의 내부 동작이 세션 .jsonl 파일 기반 |
| Level 2 | 자동 메모리 = CLAUDE.md 계층의 6번째 |
| Level 4 | `/compact`, `/resume`, `/context`의 동작 원리 |
| Level 5 | `Esc Esc` = 체크포인트 기반 되감기 |

## 기억할 것

1. **세션 = .jsonl 파일** — 프로젝트별로 저장되며 /resume으로 재개
2. **자동 압축** — 컨텍스트 가득 차면 자동 발생, 핵심 정보 보존
3. **수동 압축 (/compact)** — 포커스 지시어로 보존할 내용 제어 가능
4. **자동 메모리** — MEMORY.md 200줄이 매 세션 시작 시 로드
5. **되감기 (Esc Esc)** — git 체크포인트 기반, 대화+코드 함께 복원
6. **세션 이름 (/rename)** — 중요한 세션은 이름을 지어두면 나중에 찾기 쉬움
