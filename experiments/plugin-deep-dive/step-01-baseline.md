# Step 1: Claude Code 기본 도구 — 플러그인 없이 뭘 할 수 있나

## 핵심 질문

> 플러그인을 이해하려면, 먼저 "플러그인 없는 Claude Code"가 뭘 할 수 있는지 알아야 한다.

## Claude Code 기본 도구

| 도구 | 하는 일 | 예시 |
|------|---------|------|
| `Read` | 파일 읽기 | 소스 코드, 설정 파일 내용 확인 |
| `Write` | 파일 만들기 | 새 파일 생성 (덮어쓰기) |
| `Edit` | 파일 수정하기 | 기존 파일의 특정 부분만 변경 |
| `Bash` | 터미널 명령어 실행 | npm install, git commit 등 |
| `Glob` | 파일 이름으로 찾기 | `**/*.tsx` 패턴으로 파일 검색 |
| `Grep` | 파일 내용에서 검색 | 특정 함수나 변수 이름 찾기 |
| `WebFetch` | 웹 페이지 가져오기 | URL의 내용을 읽어오기 |
| `WebSearch` | 웹 검색 | 최신 정보 검색 |
| `Agent` | 서브에이전트 생성 | 독립적인 작업을 별도 에이전트에 위임 |

이게 전부다. 파일 읽고, 쓰고, 검색하고, 명령어 실행하는 것.

## 기본 도구로 못하는 것들

| 못하는 것 | 왜 못하나 | 어떤 플러그인이 해결하나 |
|-----------|----------|----------------------|
| 코드 타입 정보 확인 (함수 시그니처, 변수 타입) | LSP 서버 연결 도구가 없음 | OMC (lsp_hover, lsp_goto_definition) |
| 코드 구조 패턴 검색/치환 | AST 분석 도구가 없음 | OMC (ast_grep_search, ast_grep_replace) |
| 프로젝트 정보 기억 (기술 스택, 컨벤션) | 세션 끝나면 다 잊음 | OMC (project_memory_read/write) |
| 작업 중 메모 유지 | 컨텍스트 압축되면 사라짐 | OMC (notepad_write_working) |
| 여러 에이전트를 팀으로 운영 | Agent 도구는 1개만 생성 | OMC (team 시스템) |
| 작업 규율 강제 (테스트 먼저 등) | Claude는 시키는 대로만 함 | Superpowers (TDD, 디버깅 스킬) |
| 자연어 키워드로 모드 전환 | "autopilot" 같은 단어 인식 불가 | OMC (keyword-detector Hook) |

## 핵심 정리

```
Claude Code 기본 = 파일 조작 + 터미널 + 검색 + 서브에이전트

플러그인이 추가하는 것:
  - 새로운 도구 (MCP 서버)
  - 자동 실행 로직 (Hook)
  - 작업 지침서 (Skill)
```

## 확인 질문

1. Claude Code가 기본으로 제공하는 도구는 몇 개인가?
2. 기본 도구만으로 "함수의 타입 정보"를 확인할 수 있나?
3. 플러그인이 기능을 추가하는 3가지 방법은?

→ 답: 9개 / 없다 (Bash로 tsc 등 우회는 가능하지만 실시간 LSP는 불가) / MCP, Hook, Skill
